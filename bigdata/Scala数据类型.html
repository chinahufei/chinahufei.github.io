和Java访问权限的异同

Scala访问修饰符基本和Java一样，分别有：private、protected、public
如果没有指定访问修饰符，默认情况下，Scala对象的访问级别都是public
Scala中的private限定符，比Java中更严，在嵌套情况下，外层类甚至不能访问被嵌套的类的私有成员。

  • 私有成员 private (new Inner).f()访问不合法是因为f是在Inner类中被申明为private, 而这种访问需要在Inner类中
    嵌套类的private修饰访问的，需要在类中。而Java中这种可以访问。
  • 保护成员 protected
    package p{
    class Super{
    protected def f() {println("f")}
    }
    class Sub extends Super{
    f()
    }
    class Other{
    (new Super).f() //错误
    }
    }
    (new Super).f()的访问不合法是因为Scala中protected修饰的只允许自己类和子类访问，其他类不可以访问。
    而Java中，只要是同一个包的都可以访问
  • 公共成员 public
    public和Java中一样，都可以访问
  • 作用域保护
